Built under real scrutiny, not rubber-stamped: this spec went through three rounds of external critique. Two real mistakes got caught and fixed rather than left in place — the SEO content was originally targeted at homeowners instead of the actual buyer (roofing company owners), and an early cost claim said Gemini's API was free when live research showed grounding is billed separately and the free-tier terms changed twice in 2026. Both corrections are documented in the spec itself, not quietly erased.

What's Decided

  • One audience, one offer: Indianapolis-area roofing company owners, converting into booked strategy calls — not a generic multi-industry site.
  • The core mechanic: a "Gemini Visibility Check" — a real, live AI-lookup assessment, not a self-reported questionnaire dressed up as a measurement. Five versioned search queries, real scoring math, business-matching rules to prevent false-positive results.
  • Attribution-safe funnel: every lead is captured on Horsepower's own infrastructure — form, backend, Sheet — before any handoff to LaunchReady. No CTA ever redirects raw traffic off-domain.
  • Tech stack: static HTML/CSS/JS, Vercel hosting (recommended), the existing Apps Script + Sheets lead-router pattern extended with new Gemini API scoring logic.
  • Accessibility & display: WCAG 2.2 AA as the floor, large-font/high-contrast defaults sitewide, matching Jeff's standing display preference.

What's Still Open

  • Gemini proof-of-concept — real API cost, grounding availability, and latency need to be measured against an actual key and the real query set before the architecture and budget lock. Not resolvable by more research.
  • Hosting, brand assets, pricing display, and booking-link confirmation — quick decisions, still pending Jeff's sign-off.
  • Deferred to v4 on purpose: privacy/consent copy, full analytics event implementation, security/abuse enforcement detail, a functional test plan, and a deployment/rollback plan. Scoped in the spec, not yet written — sequenced this way to avoid rushing them to the same shallow-precision problem already corrected twice.
