Deep Dive · Category 04

AI Automation Layer

Three functions, unequal builds: Market is a weekly loop, Promote is build-once, Evaluate is built like software. Together they make twelve brands runnable on ~30 minutes of human time a week.

Function 1 — Market: The Weekly Content Engine

What it produces: every Monday by 9am, one batch — four sector emails plus 12–16 social posts across the brand accounts — drafted, claims-checked, and sitting in an approval doc.

INPUTS                          GENERATE              APPROVE            SHIP
─ Brand voice files (×12)   →   AI batch run,      →  Approval doc    →  Klaviyo API (emails)
─ Sector content calendar        Sunday night          approve / edit     Scheduler (social)
─ Last week's performance                              ~30 min, once
─ Claims guardrail file

The four inputs are the actual work, and they're one-time assets:

  1. Brand voice files — a half-page per brand: tone, vocabulary, banned words, the customer it talks to. This is what keeps a house of brands from collapsing into one AI-flavored voice.
  2. Sector content calendars — a 12-week rotating theme spine per sector (education / social proof / offer / founder story, repeat). The AI fills the spine; the spine stops the content from drifting random.
  3. Performance feedback — the generator reads last week's numbers from the Evaluate layer, so the loop learns instead of just producing.
  4. The claims guardrail — a standing compliance file (allowed structure/function phrasing, banned disease claims, per-SKU sensitivities) that every draft is checked against before human review. For this catalog, this is the difference between an automation asset and a regulatory liability generator.

Human-in-the-loop, deliberately: one ~30-minute approval session per week. The human never writes — they edit. That's the ten-hours-to-thirty-minutes compression, and the thirty minutes should never be automated away for a supplements catalog.

Stack cost: AI API ~$5–15/mo at this volume + scheduling tool $0–30/mo ≈ ≤$45/month.

  • Suppression hierarchy: a customer in post-purchase doesn't also get the weekly promo email — configured, not assumed.
  • Skin rule: every flow email carries the brand skin of the product bought, but the store's footer — the same constant/variable split as the product pages.
  • Ongoing AI touch is quarterly, not weekly: pull per-email revenue, draft challenger variants for the two weakest emails, A/B them. Marginal cost: $0/month.

The Monday summary format (AI-written, rigid template — numbers lead):

Store total and week-over-week move → one line per sector (revenue, the notable number, one-clause explanation) → the cross-sell path of the week and what's being done about it → one decision needed, and only when there is one. That last line is what makes it get read — it's not a report, it's a lever.

Cost: ~$1–5/month (hosting free tier + AI API cents). On ad-spend automation: the ads APIs require app review and token maintenance — weeks of friction for one number. V1 uses a spreadsheet cell per sector per week; automate in v2 only if it proves annoying.

Build Order

Weeks 1–2: Klaviyo flows + voice files → Weeks 3–4: content engine v1 running on the pre-launch list → Weeks 5–6: dashboard skeleton on real test data → Launch week: everything already running. The store opens with its nervous system installed, not bolted on later.

Total AI-layer stack cost: ~$50/month — a number worth holding next to what a part-time marketing hire costs. A full brand portfolio, thirty minutes of human time a week.

What the AI Layer Gains at Each Revenue Gate

The three functions run in full from Gate 0 — Klaviyo's day-one presence is what makes that true. As the revenue-gated stack unlocks (see the Tech Stack deep dive), the AI layer's reach grows without redesign:

  • Gate 0: full Market/Promote/Evaluate loop live — Shopify, Klaviyo, GA4, dashboard, OuterSignal VIP flagging.
  • Gate 1: OuterSignal Starter feeds real personas and demographics into Klaviyo segments — the Monday summary's sector personas switch from inference to data.
  • Gate 2: Gorgias arrives — AI ticket triage and drafted support replies join the loop; Klaviyo SMS adds a second automated channel (clean SKUs only).
  • Gate 3: Triple Whale's attribution data flows into the Monday summary (real CAC per channel replaces the spreadsheet cell); HubSpot adds wholesale pipeline status to the scoreboard.
  • Gate 4: the same engine reports D2C, wholesale, and marketplace in one summary — no new architecture, just more inputs.

The AI-Operable Rule (New in v2)

The standing selection rule for the entire stack: no tool enters unless the AI layer can drive it through an API. Every layer has been audited — Shopify (Admin API), Klaviyo (full REST), Judge.me, Gorgias, GA4, GoAffPro, and the dashboard are all directly drivable; OuterSignal's enrichment is read through Klaviyo profile properties; social posting runs through the Meta Graph API rather than closed schedulers. At setup, one credentials checklist is collected — from that day, the engine reads and acts on every layer of the store. This is what makes "market, promote, and evaluate in an automated fashion" literal instead of aspirational.
